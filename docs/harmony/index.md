# 旋律、和弦、和声

音乐不是只有单个音，还包含音与音之间的关系。这些关系分两个方向：**横向**和**纵向**。

## Melody / 旋律

旋律表示：**一个接一个出现的音，形成一条线**。

最重要的特点：**横向、按顺序展开**。

你能哼出来的那条"主线"，很多时候就是旋律。

用程序员视角 — 旋律像**数组**：

```
[C4, D4, E4, G4]     ← 强调顺序
```

## Chord / 和弦

和弦表示：**多个音同时响**。

```
C + E + G     ← 三个音同时发声
```

先别追求复杂命名，只记：**和弦 = 同时出现的多个音**。

用程序员视角 — 和弦像**集合**：

```
{C4, E4, G4}     ← 强调同时
```

## Harmony / 和声

和声比和弦更大一层。

粗略区分：

| 概念 | 本质 |
|------|------|
| 和弦 | 某一刻的同时关系 |
| 和声 | 一整段同时关系的系统 |

用程序员视角 — 和声像**状态推进**：

```
[{C,E,G}, {F,A,C}, {G,B,D}, {C,E,G}]     ← 强调整体变化
```

和声回答的问题不是"这一刻同时响什么音"，而是"这些和弦如何推进、如何回到'家'"。

## 三者的关系

```
旋律：横向的线     ─────────────→
和弦：纵向的块     ■
和声：纵向块的推进  ■ → ■ → ■ → ■
```

大部分音乐是三者的结合：旋律在和弦上方流动，和声推动整体前进。
